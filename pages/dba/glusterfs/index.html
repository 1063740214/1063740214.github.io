    GlusterFS部署

    # 一、项目配置

    本篇教程在示例步骤中使用了以下版本的软件。操作时，请您以实际软件版本为准。

    • 操作系统：CentOS 7.9 64位
    • glusterfs：9.2-1

    # 参考文档：

    官网: https://docs.gluster.org/en/latest/Quick-Start-Guide/Quickstart/ 官方安装url: https://wiki.centos.org/SpecialInterestGroup/Storage/gluster-Quickstart

    # 二、安装GlusterFS

    # 使用Storage SIG Yum Repos

    yum install centos-release-gluster -y //这个源会默认选择Gluster提供长期维护的版本
    1

    Gluster提供称为“长期维护”（LTM）和“短期维护”（STM）的不同发布流。CentOS Extras中有几个centos-release-gluster软件包，默认情况下，安装centos-release-gluster时会选择最新的LTM版本。有关不同Gluster版本的更多详细信息，请参阅发布计划。

    # 第1步 - 至少有两个节点（复制模式最好3个节点以上，防止脑裂）

    • 名为“server1”和“server2”的两台服务器上的CentOS 7
    • 一个有效的网络连接
    • 至少两个虚拟磁盘，一个用于OS安装（sda），另一个用于服务GlusterFS存储（sdb）。这将模拟真实环境部署，您可能希望将GlusterFS存储与操作系统安装分开。

    注意：GlusterFS将其动态生成的配置文件存储在/var/ lib/glusterd中，如果在任何时间点GlusterFS无法写入这些文件，它将至少导致系统性能不稳定，或者更糟糕的是使系统完全脱机。建议为/var/log等目录创建单独的分区，以确保不会发生这种情况。

    #ln -s /data/glusterfs1/etc/glusterfs/  /etc/glusterfs
ln -s /data/glusterfs1/var/lib/glusterd/ /var/lib/glusterd
ln -s /data/glusterfs1/var/log/glusterfs/ /var/log/glusterfs

    # 第2步 -格式化并安装bricks（每个节点上）

    注意：这些示例将假设brick将挂载到 /dev/sdb1

    mkfs.xfs -i size=512 /dev/sdb1
mkdir -p /bricks/brick1
    1
    2
    vi /etc/fstab
##添加以下内容：
/dev/sdb1 /bricks/brick1 xfs defaults 1 2
    1
    2
    3
    ##然后只执行挂载：
mount -a && mount
##您现在应该看到sdb1挂载在  /bricks/brick1

    注意：在CentOS 6上，您需要安装xfsprogs包才能格式化XFS文件系统 yum install xfsprogs

    # 第3步 - 安装GlusterFS（在每个节点上）

    ##安装软件（官网）：
yum install glusterfs-server -y
    1
    2

    注意：安装软件（在docker容器 需加参数--privileged=true并安装glusterfs，glusterfs-fuse；glusterfs和glusterfs-fuse为glusterfs-server的依赖包）： yum -y install glusterfs glusterfs-fuse glusterfs-server

    ##启动glusterfs：
systemctl start glusterd
systemctl status glusterd
systemctl enable glusterd

systemctl start glusterfsd
systemctl status glusterfsd
systemctl enable glusterfsd

    # 步骤5 - 配置信任池(任意一个节点上)

    ##如果在“server1”上执行
gluster peer probe server2
##注意：使用主机名时，需要从另一台服务器探测第一台服务器以设置其主机名。

##如果在“server2”上执行
gluster peer probe server1
##注意：建立此池后，只有受信任的成员才能将新服务器探测到池中。新服务器无法探测池，必须从池中进行探测。

gluster peer detach 192.168.100.111 //移除节点detach 192.168.100.111
##查看池中节点
gluster peer status //查看池中节点

    # 步骤6 - 设置GlusterFS卷

    ##在server1和server2上：
mkdir /bricks/brick1/gv0

##从任何单一服务器：
gluster volume create test-volume replica 2 server1:/bricks/brick1/gv0 server2:/bricks/brick1/gv0 force
gluster volume start test-volume

    例：

    gluster volume create kafka_logs_90_1 replica 2 192.168.100.111:/data/glusterfs1/IP_100_90/kafka_90_1/brick1/logs 192.168.100.195:/home/glusterfs2/IP_100_90/kafka_90_1/brick1/logs force
gluster volume start kafka_logs_90_1

    2
    ##/启/停/删除卷
gluster volume start test-volume
gluster volume stop test-volume
gluster volume delete test-volume
gluster volume list              /*列出集群中的所有卷*/

##查看卷是否启动（任意服务器）
gluster volume info
##注意：如果未启动卷，则会在一个或两个服务器上的/var/log/glusterfs下的日志文件中提供有关出错的线索- 通常在etc-glusterfs-glusterd.vol.log中，日志名字根据存储目录来的

gluster volume status

    # 步骤7 - 挂载GlusterFS卷（客户端操作）

    # 客户端安装
    yum install glusterfs glusterfs-fuse attr -y
    1
    # glusterfs方式挂载
    mount -t glusterfs server1:test-volume /data/test
mount -t glusterfs 服务器名/IP:卷名 要挂载的目录
df -h //查看是否挂载成功

    # 自动挂载
    ##要安装卷，请编辑/ etc / fstab文件并添加以下行：
##格式：HOSTNAME-OR-IPADDRESS:/VOLNAME MOUNTDIR glusterfs defaults,_netdev 0 0

vim /etc/fstab
server1:/test-volume /data/test glusterfs defaults,_netdev 0 0

mount -a
